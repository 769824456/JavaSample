静态工厂方法和构造方法相比
- 优势：
1. 名称易于阅读；
2. 不必在每次调用他们的时候创建对象；
3. 可以返回 原返回类型的任意子类型的对象；
4. 创建参数化类型的实例的时候，使得代码更加简洁；

- 缺点：
1. 类如果不含public 或 protect 类型的构造方法，就不能子类化；
2. 与其他静态方法没有实质上的区别；

惯用名称：
getInstance()
newInstance()
newType()
getType()
of()
ofValue()

---
总结 ：
    静态工厂方法和构造方法各有各的长处，需要考虑各自的长处。通常情况下，静态工厂方法更加合适。
